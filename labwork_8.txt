Работа с Docker (Ограничение ресурсов и работа с томами)

Выполнены практические задания по запуску и исследованию контейнеров с ограничением ресурсов и постоянным хранением данных:

- Запущен контейнер alpine с ограничением памяти 100MB.
- Исследован контейнер с ограничением CPU до 25% при выполнении скрипта cpu-stress.sh.
- Определено количество сетевых интерфейсов при обычной изоляции и при использовании сетевого пространства хоста.
- Запущен контейнер nginx:alpine в detached mode для целей профилирования и отладки.
- Изучено размещение данных Docker-контейнеров и образов в /var/lib/docker.
- Развернут контейнер MySQL с сохранением данных в подключённом томе /opt/data, обеспечивающем постоянное хранение данных даже после уничтожения контейнера.

Вывод

Цель лабораторных работ достигнута. Получены практические навыки по ограничению ресурсов контейнеров, использованию namespaces и cgroups, управлению сетевой изоляцией, а также организации постоянного хранения данных с помощью томов Docker.
