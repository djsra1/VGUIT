Отчёт по работе с индексом в Git:

1. Перешли в директорию репозитория: cd code-user

2. Создали и изменили несколько файлов:
   - INFO.md
   - README.md
   - TODO.md

3. Проверили состояние репозитория командой git status

4. Добавили один файл в индекс (git add INFO.md) и закоммитили его

5. Добавили оставшиеся файлы в индекс (git add README.md TODO.md) и сделали коммит

6. Создали дополнительные изменения в INFO.md и README.md

7. Просмотрели различия с помощью git diff

8. Использовали интерактивное добавление git add -i для выбора конкретных изменений

9. Закоммитили только выбранные изменения

10. Проверили историю коммитов с помощью git log --oneline

Вывод: работа с индексом позволяет точно контролировать, какие изменения попадут в коммит. Можно добавлять отдельные файлы, все сразу или только выбранные куски изменений.
