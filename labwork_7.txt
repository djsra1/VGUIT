Работа с Docker (ограничение ресурсов и namespaces)

Выполнены практические задания по запуску контейнеров с ограничением ресурсов и исследованию namespaces:

- Запущен контейнер alpine-100mb с ограничением памяти до 100MB.  
- Исследован скрипт /root/cpu-stress.sh, определено, какой контейнер был ограничен примерно 25% процессорного времени.  
- Запущен контейнер с командой ip addr show для подсчёта сетевых интерфейсов в обычном режиме и с использованием NET-пространства хоста.  
- Запущен контейнер http из образа nginx:alpine в detached mode с сохранением изоляции namespaces.

Вывод

Цель лабораторной работы достигнута. Получены практические навыки по ограничению ресурсов контейнеров с помощью cgroups, работе с namespaces и безопасному запуску сервисов в Docker, что важно для управления ресурсами и безопасностью при эксплуатации контейнеризованных приложений.
